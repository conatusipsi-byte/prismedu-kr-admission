## 1. Site Identity Diagnosis

- **URL**: https://conatusipsi.com/
- **확인한 페이지**: `/`, `/admissions`, `/admissions/postech/cs`, `/admissions/pusan/info-comp` (모두 404), `/pricing`, `/login`, `/signup`, `/onboarding`, `/help`, `/changelog`, `/terms`, `/privacy`, `/refund`, `/about` (404), `/blog` (404), `/contact` (404)
- **시도한 핵심 플로우**:
  - 학과 검색 → 학과 상세: ❌ **완전 차단 (404)**
  - 로그인 폼 검증(빈값/잘못된값): ⚠️ HTML5 native validation만, 커스텀 에러 메시지 없음
  - 회원가입 폼: ⚠️ 약관 동의 required=false
  - 온보딩(프로필 작성): ✅ 비로그인 접근 (보호 누락 가능성)
  - 다크/라이트 토글: ✅ 정상
- **뷰포트**: desktop(1440) 확인 완료 / mobile/tablet은 환경 한계로 emulation 대신 JS 측정 (winW=1541 고정)
- **추정 카테고리/업종**: 한국 대학 입시 컨설팅 AI SaaS (학과 단위 합격 가능성 예측 + AI 카운슬러 챗 + 시즌 단위 구독)
- **추정 타깃 유저**: 고1~고3 학생 본인, 학부모, 재외국민 특례 지원자, 특히 정량 데이터 기반 의사결정을 원하는 보수적 학부모
- **현재 표방하는 톤(추정)**: "정직한 데이터/모던 SaaS/약간의 뉴-에디토리얼" — 진한 헤딩 + 청록 그라데이션 강조 + Pretendard 기반 미니멀
- **이 톤이 비즈니스에 적합한가**: 적합도 ⭐⭐⭐⭐☆. "표본 부족 학과는 비공개"라는 "정직성" 메시지와 차분한 톤이 일치. 다만 입시 시장 학부모는 신뢰 시그널(사업자번호/실명 후기/연락처)이 매우 중요한데 푸터에 "사업자등록번호 — 출시 직전 등록 예정" 그대로 노출되어 있어 신뢰도 가산이 못 일어남.

## 2. Theme & Direction Fit (총평)

- **디자인-비즈니스 톤 일치도: ⭐⭐⭐⭐☆ (4/5)** — 컬러(emerald/teal primary `hsl(160 84% 39%)` ≈ `#10B981`), Pretendard Variable, 부드러운 radius, 카드 기반 레이아웃, 다크모드 지원까지 일관성은 매우 양호. 다만 콘텐츠/기능 미완성이 톤을 흔든다.
- **가장 큰 톤 충돌 요소 Top 3**:
  1. **푸터 "출시 직전 등록 예정"** 문구를 그대로 공개 — "정직한 데이터" 메시지와 충돌(미완성 노출).
  2. 학과 카드 클릭 시 **거의 절반이 404로 떨어짐** — "1,000+ 학과 데이터"라는 슬로건과 모순.
  3. CTA 섹션(`#0f1729` 어두운 배경 위 흰색 "무료로 시작하기" 버튼은 `bg-white text-ink-950`) — 흰 버튼이 메인 primary와 시각적 위계 충돌, hero의 `bg-brand-600` 버튼과 스타일 통일성 깨짐.
- **차별성/기억 잔존도**: "3초면 알 수 있어요" + "모르는 건 모른다고 말합니다" 카피의 인상은 강함. Reach/Match/Safety 분류 시각화도 카테고리에서 새로움.

## 3. Typography Audit

- **사용 중인 폰트**:
  - Body: `"Pretendard Variable", Pretendard, -apple-system, BlinkMacSystemFont, system-ui, Roboto, sans-serif`
  - H1: `"Pretendard Variable", Pretendard, "Inter Variable", Inter, sans-serif`
  - 영문 본문: Inter Variable (latin subset 로드 확인)
- **개수**: 2 폰트 패밀리 = 적정.
- **톤 적합성**: ✅ Pretendard는 모던/중립/SaaS·테크 톤에 정확히 매치. Inter도 잘 어울림.
- **발견된 타입 이슈**:
  - H1: `font-size: 72px`, `line-height: 73.44px`, `letter-spacing: -2.88px` — line-height 1.02는 한글 큰 헤딩에서 위/아래 받침이 다음줄과 부딪힐 위험. **권장: line-height 1.1~1.15** (헤딩 1.1–1.3 권장 범위 하단). 모바일에서 자동 축소 안 되면 줄바꿈 깨짐 위험.
  - 모바일에서 H1 72px 그대로 유지 시 너무 큼 — clamp() 적용 필요(확인 필요).
  - Pretendard Variable subset 11개(82~91) 모두 첫 로드에 강제 다운로드되어 폰트 트래픽 비중이 큼.

## 4. Color System Audit

- **추출 팔레트(CSS variables, HSL)**:
  - `--background`: `hsl(210 40% 98%)` ≈ `#F8FAFC`
  - `--foreground`: `hsl(222 47% 11%)` ≈ `#0F1729`
  - `--primary`: `hsl(160 84% 39%)` ≈ `#10B981` (emerald)
  - `--ring`: 동일 emerald
  - `--accent`: `hsl(160 70% 95%)` (mint pastel)
  - `--success`: `hsl(142 71% 36%)` ≈ `#1A9F4D`
  - `--warning`: `hsl(38 92% 45%)` ≈ `#DC8B0A`
  - `--destructive`: `hsl(0 84.2% 60.2%)` ≈ `#EF4444`
  - `--muted`: `hsl(210 20% 92%)`
  - `--border`: `hsl(214 32% 91%)`
- **체계성**: ✅ shadcn/tailwind 표준 토큰 체계. Reach=red, Match=blue, Safety=green semantic 사용 일관됨.
- **톤 무드**: 차가운 톤 (cyan-emerald 그라데이션) + 학구적 묵직함. 입시 도구에 적합.
- **대비비 위반(가시 의심)**:
  - 헤더의 muted 텍스트(`text-foreground/60` 추정) + 반투명 배경(`bg-background/55`) 조합 — 측정 필요.
  - 다크모드 CTA 섹션 본문 카피 회색(`text-white/70` 추정) 위 `text-base`: 가시성 낮음.
  - "표본 부족 → 확률 비공개" 옅은 yellow tint 박스 안 텍스트(`#fbe5b3` 비슷) — 확인 필요.
- **개선 제안**: 라이트모드의 muted-foreground 토큰을 1단계 짙게(`hsl(215 16% 35%)`로) 조정.

## 5. Functional Health Check (요약)

- **핵심 플로우 완주율**: 1/3
  - ✅ 다크모드 토글, 정적 페이지 열람
  - ❌ 학과 검색 → 학과 상세: **404 (라우트 또는 데이터 매핑 실패)**
  - ❌ 회원가입/로그인 E2E: 시도하지 않음(소셜·이메일 모두 외부 의존 + DB 부재 가능성). 약관 동의 체크박스 미필수 자체가 결함.
- **Console 에러 개수**: 추적 시작 시점 이후 0건 (페이지 새로고침 후 별도 확인 필요)
- **Network 4xx/5xx 개수**:
  - **503 Service Unavailable**: `/admissions/pusan/info-comp`, `/admissions/postech/cs`, `/admissions/sogang/computer-science`, `/admissions/korea/design-art` (4건) — 일부는 재요청 200, **불안정한 서버 응답**.
  - **404 Not Found**: `/about`, `/blog`, `/contact` (푸터에서 링크되어 있음)
- **깨진 링크 개수**: 최소 3건 (about/blog/contact) + 학과 상세 페이지 16건 중 다수
- **Critical 기능 버그**: 학과 검색→상세 라우팅 무너짐(P0).

## 6. P0 — 즉시 수정 (출시/공개 전 반드시)

### [P0-01] 학과 검색 결과 카드 클릭 시 404 (핵심 기능 마비)
- **위치**: `/admissions` → 카드 → `/admissions/[universityId]/[departmentId]`
- **추정 파일 경로**: `app/admissions/[universityId]/[departmentId]/page.tsx`, mock-data
- **카테고리**: M, X, Q (네비게이션/라우팅/핵심 플로우)
- **유형**: 기능
- **재현 단계**:
  1. https://conatusipsi.com/admissions 진입
  2. 결과 카드 중 "POSTECH 컴퓨터공학과" 또는 "부산대 정보컴퓨터공학부" 클릭
  3. URL: `/admissions/postech/cs` 또는 `/admissions/pusan/info-comp`로 이동
- **기대 결과**: 학과 상세 페이지(전형, 합격 가능성, AI 분석)가 렌더링
- **실제 결과**: H1="존재하지 않는 학과입니다", title="학과를 찾을 수 없습니다", 본문에 "404"가 표시됨. 네트워크 응답이 첫 요청에서 **503**, 재요청에서 200으로 들쭉날쭉.
- **수정안**:
  1. `/admissions` 리스트 API에서 반환되는 카드의 `href`와 상세 페이지 라우트 파라미터(`universityId`, `departmentId`)가 실제 데이터셋의 키와 동일한지 검증.
  2. 503 응답 SSR/RSC 페이지는 데이터 fetch 실패 시 retry-aware 에러 반환.
  3. 상세 페이지 `generateStaticParams`에 실제 존재하는 학과 id 목록만 등록.

### [P0-02] 푸터 링크 /about, /blog, /contact 모두 404
- **위치**: 푸터 "회사 소개", "블로그", "문의" 링크 (`components/nav/Footer.tsx`)
- **카테고리**: X, M, N
- **수정안**: (A) 최소 페이지 스텁 생성 OR (B) 푸터에서 링크 제거.

### [P0-03] 회원가입 약관 동의 체크박스가 필수가 아님
- **위치**: `/signup` 약관 체크박스 (`app/login/LoginView.tsx`)
- **카테고리**: R, N, 보안/법적
- **수정안**: zod 스키마 + HTML5 required + 에러 메시지.

### [P0-04] 푸터에 "사업자등록번호 — 출시 직전 등록 예정" 그대로 노출

- **수정안**: 등록 전까지 해당 라인 숨김 OR PRE-RELEASE 배너 명시.

### [P0-05] 학과 상세 라우트 일부 첫 요청 503
- **위치**: SSR/RSC endpoint `/admissions/{uni}/{dep}?_rsc=...`
- **카테고리**: W, O
- **수정안**: ISR + retry + Sentry 추적.

### [P0-06] CTA 섹션 흰색 버튼 위계 충돌 + 그라데이션 텍스트 가독성

- **카테고리**: C, D, F
- **수정안**: 버튼 variant 통일, 클래스 충돌 정리.

### [P0-07] 로그인 폼에 커스텀 에러 메시지/실시간 검증 없음

- **카테고리**: R
